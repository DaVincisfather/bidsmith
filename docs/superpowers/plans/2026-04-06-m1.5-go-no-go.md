## Step 2: Database migration

**File:** `supabase/migrations/003_go_no_go.sql`

```sql
create table go_no_go_assessments (
  id uuid primary key default gen_random_uuid(),
  analysis_id uuid references analyses(id) not null,
  organization_id uuid references organizations(id),
  team_consultant_ids uuid[] not null,
  result jsonb not null,
  decision text not null default 'pending' check (decision in ('pending', 'go', 'no-go')),
  decision_at timestamptz,
  created_at timestamptz default now() not null
);

create index idx_go_no_go_analysis on go_no_go_assessments(analysis_id);
create index idx_go_no_go_org on go_no_go_assessments(organization_id);
```

## Step 3: Go/No-Go evaluator (core AI logic)

**File:** `src/lib/go-no-go-evaluator.ts`

Single function: `evaluateGoNoGo(analysis, teamConsultants, allScoredConsultants) → GoNoGoResult`

- Takes RfpAnalysis, the locked team (Consultant[]), and full scored list (ScoredConsultant[])
- Single Sonnet call with structured prompt
- Prompt instructions:
  1. Check each must-requirement against team — binary pass/fail
  2. If any must-req fails → winProbability = 0
  3. Assess coverage of should/nice-to-have requirements
  4. Weight evaluation criteria
  5. Factor in red flags
  6. Generate improvement suggestions comparing team gaps against pool
  7. Return structured JSON
- Parse and return GoNoGoResult

**Test:** Unit test with mocked Anthropic response.

## Step 4: API route

**File:** `src/app/api/go-no-go/route.ts`

`POST /api/go-no-go`

**Input body:** `{ analysisId: string, teamConsultantIds?: string[] }`

**Logic:**
1. Fetch analysis from DB
2. Fetch all consultants for org (same pattern as matches route)
3. Fetch latest match result for the analysis (scored consultants)
4. If teamConsultantIds omitted → pick top-scored per level from match
5. Call `evaluateGoNoGo()`
6. Save to `go_no_go_assessments` table
7. Return result

**Also:** `PATCH /api/go-no-go/[id]/route.ts` — update decision (go/no-go)
